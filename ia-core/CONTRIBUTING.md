# Guia de Contribuição

1. Siga o padrão de código definido pelo Checkstyle.
2. Escreva testes automatizados para novas funcionalidades.
3. Atualize a documentação (README.md) quando necessário.
4. Execute `./check_project.sh` antes de enviar PRs.
5. Use nomes de classes, métodos e variáveis em inglês.
6. Centralize dependências no parent POM.
