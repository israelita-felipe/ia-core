PARTE 2: ANTI-PATTERNS E DESIGN SMELLS IDENTIFICADOS
================================================================================

[AS-1] GODCLASS PATTERN: DefaultBaseService (181 linhas)
- Implementa 5 interfaces, violando SRP
- Solução: Usar padrão Mediator ou Facade para delegação explícita

[AS-2] TEMPORAL COUPLING: PessoaService.synchronize() 
- Lógica dependente da ordem de execução
- Solução: Usar JPA cascade operations ou event-driven architecture

[AS-3] MIXED CONCERNS: DefaultSecuredBaseService
- Segurança misturada com lógica de negócio
- Solução: Usar decorador ou AOP para aspectos de segurança

[AS-4] FEATURE ENVY: SaveBaseService.save()
- Método chama muitos métodos de outras classes
- Solução: Refatorar em classes menores ou usar padrão Command

[AS-5] MAGIC STRINGS: EventType values hardcoded
- Mudanças espalhadas pelo código
- Solução: Centralizar enum EventType com validação

[AS-6] NULL POINTER RISKS
- Múltiplas classes retornam null sem documentação
- Solução: Usar Optional<D> ou Result<D> monads

[AS-7] TIGHT COUPLING TO FRAMEWORK
- Dependência direta de Spring em AbstractBaseService
- Solução: Usar inversão de controle com interfaces customizadas
